# 🏢 企业资产管理领域知识

## ⚠️ 使用范围

**本技能专门用于 `/speckit.specify` 阶段。**

### 为什么只用于 specify 阶段？
**核心理念：** 领域知识用于增强需求规范，一旦 spec.md 生成完整，后续阶段直接依赖 spec.md 即可。

---

## 🎯 在 Specify 阶段的作用

### Claude 必须做的事：

1. **加载领域知识**
```
   在用户执行 /specify 后，立即阅读：
   references/domain-knowledge.md
```

2. **理解核心概念**
   
   - 资产生命周期（库存中→已分配→维修中→已报废）
   - 状态转换规则（什么状态可以转换到什么状态）
   - 关键业务约束（如：只有"库存中"才能分配）
   
3. **增强用户需求**
   用户可能只说："我要做资产分配功能"
   
   Claude 应该基于领域知识自动补充：
   - "资产分配前需要验证状态吗？"（领域知识：必须验证）
   - "如果资产在维修中怎么办？"（领域知识：禁止分配）
   - "需要记录分配历史吗？"（领域知识：必须记录审计日志）
   - "性能要求是什么？"（领域知识：5万数据<200ms）

4. **生成完整的 spec.md**
   
   确保claude读取了相关的领域知识，然后再继续`/specify`命令的流程

---

## 📋 使用步骤

### 步骤 1：用户开始需求定义
```bash
用户：/specify
我要开发资产分配功能，让管理员能把笔记本分配给员工
```

### 步骤 2：Claude 激活领域知识
```
Claude 内部流程：
1. 检测到 /specify 命令
2. 自动阅读 references/domain-knowledge.md
3. 理解其中的核心概念和规则
```

### 步骤 3：Claude 基于领域知识对用户输入中不明确的点自动进行补充

比如用户输入可能只写了："我要做资产分配功能"，此时 Claude 就应该基于领域知识库自动对用户输入进行补充，例如性能要求、需要记录审计日志，资产分配前现需要验证状态等等内容。

### 步骤 4：基于增强后的用户输入执行/specify流程生成 spec.md

将补充增强后的用户输入作为 `/specify `命令的真实输入执行它的流程

## ✅ 成功标准

Claude 正确使用此技能的标志：

- [ ] 在 specify 阶段首先主动阅读了 domain-knowledge.md 中的领域知识，对用户输入进行了补充和增强
- [ ] 将补充增强后的用户输入作为specify命令的输入
- [ ] spec.md 足够完整，以至于后续 plan/task/implement 阶段不需要再回来查领域知识
